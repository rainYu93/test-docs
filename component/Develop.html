# 主题颜色规范

注意

背景：srm-front-app子应用服务，1.21.0及之后版本，支持主题切换，主题切换后整体的背景颜色及其配色会动态变化。 开发者在开发过程中，自定义的开发内容，如果样式颜色是需要跟随主题切换进行变动的，这就要求开发者, 自定义样式中涉及颜色的，统一使用css变量而不使用具体颜色值。

警告

注意：使用透明度的颜色，需要统一写成rgba形式，部分安卓浏览器不兼容十六进制透明度

     //bad
     color: #276EF1;
     background: #FFFFFF;
     border: 1px solid #F6F6F6;
     stroke: #333;
     // good
     color: var(--color-primary);
     background: var(--background-color-empty);
     border: 1px solid var(--split-line-color-primary);
     stroke: var(--font-color-default);

# 国际化规范

所有涉及到文字显示的地方均需要使用多语言。$T(code).$D(msg)

CODE 命名规范
CODE 分为两段

第一段：
    [服务名].[功能名]
    
    例如上述例子中: smbl.common 其中 smbl 是服务名, common 是功能名

第二段
    第二段分为 2 种类型：
    
    模型
        model.[模型名].[字段名]
        
        例如： model.common.code 是 common 模型的 code 字段
    
    视图
        view.[类型].[具体类型].[具体含义]
    
    例如：
    
        验证类型: view.validation.require
        提示信息: view.message.success
        操作类型: view.option.submitOrder
        注意：统一按照 [具体类型].[具体含义] 的方式定义
        
    例如：view.button.save, view.button.submit

* 当前页面用到的所有多语言模块需要维护到vue组件的modules下，形如：
modules: [ "ssrc.inquiryHall","hipspfm.order" ]
eslint运行时会自动修复页面里没有维护的多语言模块，开发提交代码时务必走一次githook修复没有维护的多语言模块，确保万无一失。

# 日期时间规范

前后端交互格式
前后端传递日期格式统一为：

日期(较少出现): YYYY-MM-DD
日期时间(绝大多数): YYYY-MM-DD HH:mm:ss
时间(几乎没有): HH:mm:ss

# 组件命名

新开发的组件名需要是Srm开头的大驼峰格式，组件名需要见名知意，尽量使用常见组件库的单词命名，单词使用全名。 单文件组件的文件名应该要么始终是单词大写开头 (PascalCase)

 //bad
 srmbutton.vue
 srmButton.vue
 srm-button.vue
 // good
 SrmButton.vue

# 日志规范

# 注释规范

采用jsDoc标准注释规范
    关于jsDoc 参考:
    
    http://usejsdoc.org/

方法注释：
    @async 一般情况下使用async函数需要注明
    
    @params 说明参数
    
    @returns 返回值
方法注释实例：
    /**
     * 查询 说明接口逻辑用途
     * @async
     * @function queryMembers 接口函数名称
     * @param {object} params - 接口参数
     * @param {!number} [params.page = 0] - 数据页码
     * @param {!number} [params.size = 10] - 分页大小
     * @returns {object} fetch Promise 默认返回值
     */
    
js文件注释：
    description 说明文件名称及用途
    
    @date 创建日期
    
    @author 作者
    
    @version 版本号
    
    @copyright 公司信息
    
js文件注释实例：
    /**
     * test - 测试代码
     * @date: 2018-7-24
     * @author: yourname <yourname@going-link.com>
     * @version: 0.0.1
     * @copyright Copyright (c) 2018, Zhenyun
     */

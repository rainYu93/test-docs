# 1 概述

# 1.1 文档概述

此文档将介绍钉钉集成SRM移动端的相关配置，主要包括钉钉开放平台配置和云SRM配置

# 2 配置说明

# 2.1 SRM配置

# 2.1.1 三方平台定义

菜单路径：移动端管理-三方平台-三方平台定义

若没有对应平台，需要新建一个三方平台，若已有对应平台，直接使用已定义的三方平台即可

[注：种子数据部署完成，此配置正常会自动生成 ]

  • 三方平台编码：dingdingMicroAppProxy
  • 描述：钉钉微应用-代理平台
  • 消息发送地址：https://oapi.dingtalk.com/topapi/message/corpconversation/asyncsend_v2
  • 获取用户地址：https://oapi.dingtalk.com/gettoken
  • 执行代码：此代码主要是回调三方，获取用户信息，并二次握手做安全性校验

# 2.1.2 三方平台参数定义

菜单路径：移动端管理-三方平台-三方平台定义 -》 参数定义

  • REDIRECT_URL_TEMPLATE：REDIRECT_URL_TEMPLATE&microapp=CORP_ID 跳转地址模板（构造消息跳转链接时使用）
  • TOKEN_URL：https://oapi.dingtalk.com/gettoken 获取钉钉token地址

# 2.1.3 三方平台运营号维护

菜单路径：移动端管理-三方平台-三方平台运营号维护

# 2.1.3.1 新建一个三方运营账号

  • 账号：非必输，SRM自动生成（若需要自定义，可以维护相应的值）
  • 租户：选择企业微信对应的租户 *必输
  • 三方平台：选择步骤 2.1.1 中定义的钉钉微应用-代理平台（dingdingMicroAppProxy） *必输
  • 应用：根据实际使用，选择商城或者SRM
  • 三方账号：应用AppKey【见步骤：2.2.4.1】 *必输
  • 三方密钥：应用AppSecret【见步骤：2.2.4.1】 *必输
  • 运营号描述：租户名称+钉钉微应用（例：甄云科技钉钉微应用） *必输
  • 备注：若无特别需要备注的，与运营号描述相同即可
  • 平台运营账号：默认不开启（当租户为SRM平台时，可以开启，将当前三方运营号作为平台级运营账号使用）
  • 自动绑定：默认不开启（当三方账号通过接口，在srm系统中同步生成子账号时，开启该功能可实现免登录）
  • 启用：默认开启（当三方运营号不使用时，可禁用）
  • 启用客户系统认证：默认不开启（当三方系统需要使用到自己的客户系统认证账号时，可开启该功能，并结合客户系统认证执行代码使用）
  • 客户系统认证执行代码：客户系统认证流程，当启用客户系统认证标识开启时需要维护
# 2.1.3.2 获取三方运营账号生成的账号信息，用于回调地址的构造

三方运营账号信息：xxxx

# 2.1.4 三方运营参数配置


  • proxyUrl：代理域名（客户代理域名，转发请求到钉钉，解决钉钉出口ip不能被多企业配置问题） *必输
  • AGENT_ID: 微应用AgentId（用于消息推送时，指定推送应用） *必输
  • CORP_ID：企业Corpid（用于消息跳转时，构造跳转地址） *必输
  • SEND_MESSAGE_URL：钉钉消息推送地址（若不配置，则取平台级推送地址，当使用客户代理域名时，需要单独配置租户级推送地址）
  • TODO_FLAG：推送钉钉待办标识（启用：1；不启用：0）
  • CREATE_TODO_URL：钉钉待办地址（推送工作流待办消息到钉钉待办）
  • UPDATE_TODO_URL：钉钉已办地址（推送工作流已办消息到钉钉，更新待办消息）
  • TODO_REDIRECT_URL_TEMPLATE：待办跳转模板

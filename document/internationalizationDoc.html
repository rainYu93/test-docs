# 1 概述

# 1.1 文档概述

此文档将介绍移动端国际化改造相关内容

注意

依赖srm-front-app-core，内测版本>=2.1.1-alpha或正式版本>=2.1.1时，建议文档中涉及的国际化辅助方法 1、数值根据语言环境切换分隔符：calcMultipleDelimiter； 2、获取币种对应的精度：calcCurrencyPrecision； 3、限制数字输入精度：getNumPrecisionInput 替换为srm-mobile-ui组件库中SrmUtil提供的以下三个方法 getNumberLocalized，getCurrencyPrecision，getNumPrecisionInput。

# 2.1 数值（千分符、小数点）

显示页面需要将数值使用千分符、小数点进行分隔

改造示例:

使用个性化组件，设置以下属性

  • fieldType：INPUT_NUMBER （仅当editor为false时，会对数值做分隔符处理）
  • editor：false （控制是否可编辑）
{
    fieldName: "数量",
    fieldCode: "quantity",
    langCode: "sodr.orderApproval.model.common.quantity",
    fieldType: "INPUT_NUMBER",
    editor: false,
},

# 2.2 数量精度

  • 只读页面：显示分隔符，显示后端返回值
  • 编辑页面：不显示分隔符，根据配置做精度控制，控制可输入的小数位

pc端精度配置

菜单：计量单位定义

使用示例:

取返回值中 uomPrecision 字段，即表示数量精度

注意：和相应负责人确认，是否需要根据单据来源不同，来判断单据是否要做相应的精度控制

改造示例:

(1) 显示页面

  • fieldType：INPUT_NUMBER （仅当editor为false时，会对数值做分隔符处理）
  • editor：false （控制是否可编辑）
{
    fieldName: "数量",
    fieldCode: "quantity",
    langCode: "sodr.orderApproval.model.common.quantity",
    fieldType: "INPUT_NUMBER",
    editor: false,
},

(2) 编辑页面

  • fieldType：INPUT_NUMBER （仅当editor为false时，会对数值做分隔符处理）
  • required：是否必输
  • editor：true (控制是否可编辑)
  • editLimit：true (是否启用编辑精度输入限制，当超出精度位数不可输入，仅可编辑时有效)
  • precision：输入精度，传入值需为Number类型（启用编辑时，editLimit为ture，根据精度控制输入的位数，editLimit未启用，数值进行四舍五入）
{
    fieldName: "数量",
    fieldCode: "quantity",
    langCode: "sodr.confirmOrder.model.common.quantity",
    fieldType: "INPUT_NUMBER",
    required: true,
    editor: this.editFlagFunction("QUANTITY", false), //控制是否可编辑
    editLimit:true, 
    precision: (val,record) => {
        // erp订单及电商申请转订单不做精度控制
        // erp订单：sourceCode="SRM"&&poSourcePlatform="ERP"
        const erpFlag = record.sourceCode === "SRM" && record.poSourcePlatform === "ERP";
        // 电商申请转订单：sourceCode="SAP"&&poSourcePlatform="E-COMMERCE"
        const sapFlag = record.sourceCode === "SAP" && record.poSourcePlatform === "E-COMMERCE";
        if (!erpFlag && !sapFlag) {
            return Number(record.uomPrecision);
        }
    }
},

# 2.3 金额/单价精度

pc端精度配置

菜单：计量单位定义

配置对应精度控制关系

  • 单价精度：单价精度
  • 财务精度：金额精度

获取金额/单价精度工具方法

  • /**
  • 获取币种对应的精度
  • @param {String} currencyCode
  • @param {String} precisionType
  • financialPrecision 财务精度
  • defaultPrecision 单价精度
  • **/

工具类方法：calcCurrencyPrecision（currencyCode,precisionType）

使用示例：

传入相应币种编码和金额/单价类型

注意：和相应负责人确认，是否需要根据单据来源不同，来判断单据是否要做相应的精度控制

{
    fieldName: "含税行金额",
    fieldType: "INPUT_NUMBER",
    fieldCode: "taxIncludedLineAmount",
    langCode: "sodr.orderApproval.model.common.taxedLineAmount",
    editor: false,
    precision: (val,record) => {
        // erp订单及电商申请转订单不做精度控制
        // erp订单：sourceCode="SRM"&&poSourcePlatform="ERP"
        const erpFlag = record.sourceCode === "SRM" && record.poSourcePlatform === "ERP";
        // 电商申请转订单：sourceCode="SAP"&&poSourcePlatform="E-COMMERCE"
        const sapFlag = record.sourceCode === "SAP" && record.poSourcePlatform === "E-COMMERCE";
        if (!erpFlag && !sapFlag) {
            return Number(this.$options.filters.calcCurrencyPrecision(record.currencyCode,"financialPrecision"));
        }
    },
},

  • 只读页面：显示分隔符，精度满足时显示后端返回值，精度不足时自动补0（即：精度为3时，返回 2.33 显示 2.330）

# 1 概述

# 1.1 文档概述

此文档将介绍移动端消息跳转对应界面相关配置。（仅移动端单独集成，走集中交付统一集成，此文档配置不适用）

# 2 配置说明

# 2.1 平台标准模板消息

以下配置适用于平台级的标准模板消息，配置后所有租户下的系统消息都将跳转到相应页面

前提条件：pc端已配置相应的标准模板消息，若未配置，则移动端暂时不做

# 2.1.1 登录SRM平台管理员账号

需要登录 SRM平台移动端平台租户管理员 账号（需向移动组申请）

# 2.1.2 pc端消息模板配置获取

菜单路径：消息管理-消息模板

  1. 搜索pc端对应的消息模板，这里以订单发布为例
  2. 搜索结果一般有短信、邮件、站内信三种，移动端取站内信消息模板 【例：订单发布-站内消息】
  3. 获取对应消息模板代码：SODR.ORDER_PUBLISH_NEWS （移动端消息模板配置，需要保持相同编码）
  1. 查看消息模板明细，确定模板中已经传入的参数，是否满足移动端跳转需要

若参数不满足时，需要联系对应PC后端技术，添加移动端跳转所需的参数，参数名需要和后端技术沟通后确定

# 2.1.3 移动端消息模板-子应用映射配置

菜单路径：移动端管理-消息管理-（消息模板-子应用映射配置）

新建移动端消息模板

  • 模板代码：使用pc端已配置的消息模板代码（见步骤：2.1.2）【例(订单发布-站内消息)：SODR.ORDER_PUBLISH_NEWS】
  • 推送频道：根据消息类型选择
  • 子应用地址模板：子应用地址【例：/OrderConfirmation/#/OrderConfirmationList?headerId=${headerIdEncrypt}&switchTenantId=${switchTenantId}】
  1. 使用 ${} 获取后端传过来的参数，先查看pc端模板中是否有所需参数，若无，则联系对应后端技术添加
  2. switchTenantId：该参数用于切换指定租户，若账号属于多家公司，传入该值可以指定要切换的租户，避免权限问题无法查看单据；不传该值情况下，默认是消息发送方租户tenantId
  • 备注：消息模板名称

# 2.2 租户模板消息

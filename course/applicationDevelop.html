# 标准子应用开发，建议开发步骤如下

  1. 在项目中找到路径 src/modules/ , 在该路径下以子应用名称新建一个文件夹如 Example(文件夹用大驼峰命名)。

  2. 在 Example 目录下建立如下目录

目录 描述
api 与后端交互接口定义
components 子应用私有化组件（一般不具有通用性）
mixins 子应用mixin
router 子应用路由配置（必需）
store store 配置（必需）
views 视图页面（必需）

  1. 在 views 目录下也用大驼峰命名建立 vue 文件，如 SrmExample.vue(注意编码时写注释)

  2. 状态管理配置 在子应用 store 目录下建立 index.js，并参照如下配置(注意哪怕没用到 vuex 也需要建立这个文件)

import { createGetter, createMutations } from "@/store/utils";
// 1.27.0及之后版本微服务拆分，使用下面方式引入
// const { createMutations, createGetter } = require("srm-front-app-core/script/store"); 

let state = {};

export default {
  namespaced: true,
  state,
  getters: createGetter(state),
  mutations: createMutations(state),
};

# 2.2 标准二开子应用开发

目前移动支持个性化，如果通过个性化配置即可实现，配置即可；若需开发，需要在 src/modules/CUX目录下进行文件夹的新建，文件夹名称使用租户简称作为前缀 eg：src/modules/CUX/BJT_MyOrder,其他操作参照标准子应用。 1.27.0及之后版本微服务拆分，微服务拆分后的srm-front-app-cux二开服务无需此目录， 本就是二开服务。

# 2.3 子应用配置

  1. 线上使用，需要在移动端管理下的子应用维护里配置子应用图标，类似pc菜单。

  2. 子应用维护配置

    字段 参考值 描述 是否必填
    子应用图标 - 子应用图标 必填
    子应用编码 Srm_Example 单词间以下划线分割 必填
    子应用地址 /SrmExample 以大驼峰命名，对应子应用目录；若同一个目录不同的页面入口对应多个菜单，可在原路径后拼接 #/+路由 的方式指定 必填
    子应用名称 示例页面 子应用名称 必填
    子应用描述 示例页面 子应用描述 非必填
    应用 甄采云 有甄采云和企业商城选项，这里选甄采云即可 必填
    是否外部子应用 否 是否是外部子应用，如果是外部子应用，子应用地址需要填写绝对地址。不会拼接相关域名 必填
    关联菜单 - 通过关联菜单授权，子应用权限与菜单绑定，即某角色可见绑定的菜单即可见该子应用 不必填
    授权角色 - 通过角色授权，子应用权限与角色绑定 不必填
    分组名称 敏捷协同 根据业务要求 必填
    排序 1 填数字，是 子应用列表的顺序 必填
    启用 是 是否启用子应用 必填

    基本配置这些即可，其他的看具体业务要求

# 2.4 其它

警告

标准的子应用请使用平台级租户管理员配置，其他租户可以在"子应用维护"对应功能下直接引用；二开的由项目对应租户管理员进行配置。

# 1 概述

# 1.1 文档概述

概述

移动端于2.1大版本发布后，由原单体服务拆分为所属四个套件的移动微服务，拆分后原属于一个服务的代码分属多个模块，不同模块之前的代码依赖或者联动由之前的单服务处理变为跨服务处理。 为解决此问题，我们引入了webpack5联邦模式，通过远程组件引入的方式实现跨服务联动。当前文档着重说明在本地开发时如何启动多个服务，实现跨服务联动开发。依赖最低内测版本 "srm-mobile-ui": "2.1.20-alpha", "srm-front-app-core": "2.1.11-alpha"。

# 2 开发说明

# 2.1 服务说明

我们直接以服务举例说明，srm-front-app-plat为消费者（即组件依赖者，以下简称plat），srm-front-app-sour为生产者（即组件曝露者，以下简称sour）。plat跨服务加载sour曝露的组件。

# 2.2.4、plat使用代码参考

我们提供两种远程组件加载的代码参考，方式一：我们组件库提供的动态加载远程组件，适用根据配置动态加载远程组件的场景（工作流加载表单使用的为当前方式），参数一：远程服务名称；参数二：固定default； 参数三：./曝露组件名称；参数四：远程组件js加载地址，等价2.2.3提到的配置项去掉appXXX@前缀。 方式二：vue提供的固定加载组件的方式，适用于固定的组件加载，引入方式为远程服务名称/曝露组件名称。具体代码如下：

<template>
    <div>
        <div>
            <component
                    :is="currentCom"
                    v-if="currentCom"
                    ref="childComponent"
                    msg="方式一"
                    style="width: 100%;height: 100%"/>
        </div>
        <div style="height: 50px;"/>
        <div>
            <hello-world msg="方式二"/>
        </div>
    </div>
</template>
<script>
    import { SrmUtil } from "srm-mobile-ui";
    export default {
        // 方式二
        components: {
            HelloWorld(resolve) {
                require([ "appSour/HelloWorld" ], resolve);
            }
        },
        data() {
            return {
                currentCom:null
            };
        },
        mounted() {
            // 方式一
            SrmUtil.quietlyImportComponent(SrmUtil.loadRemoteComponent("appSour", "default", "./HelloWorld", "http://localhost:9598/SourRemoteEntry.js")()).then((com) => {
                this.currentCom = com && com.default;
            });
        },
    };

</script>
